Family Data Web API Documentation

Adult

The Adult is sent back and forth as json.
The json format is seen in the example below:
{
    "JobTitle": "Tutor", 
    "ID": 0,
    "FirstName": "Aryan",
    "LastName": "Gifford",
    "HairColor": "Black",
    "EyeColor": "Brown",
    "Age": 56,
    "Weight": 99.2,
    "Height": 172,
    "Sex": "M"
}
API Request Sub-path Description Exceptions
/Adults GET /{id:int}
This request will return an Adult.
{id} is an integer.
Example:
  • .../Adults/3
  • .../Adults/7
  • .../Adults?id=5
 May return 404 Not Found, if the provided ID did not match any Adults.
This request will return a collection of Adults.
This request takes an optional number of the following request parameters:
  • firstname
  • lastname
  • jobtitle - spaces in URL are either - or %20
  • haircolor
    valid values: [brown, blond, red, black, white, grey, blue, green, leverpostej]
  • eyecolor
    valid values: [brown, grey, green, blue, amber, hazel]
  • sex:F/M
  • age
  • id
Example:
  • .../Adults
  • .../Adults?firstname=jens&haircolor=black&jobtitle=engineer
  • .../Adults?lastname=petersen&eyecolor=green
  • .../Adults?jobtitle=police-officer
  • .../Adults?jobtitle=police%20officer
 May return 500 Internal Server Error, if something fails.
PUT
Use this to create a new Adult. Returns the created Adult. Provide the Adult as json in the request body.
Example:
  • .../Adults

May return 400 Bad Request, if the data is invalid.
May return 500 Internal Server Error, if something fails when creating the adult.

DELETE /{id:int}
Use this to delete an Adult. Provide the id for the Adult to delete. Returns the deleted Adult.
Example:
  • .../Adults/5
  • .../Adults?id=3

May return 404 Bad Request, an Adult was not found with the provided id.
May return 500 Internal Server Error, if something fails when creating the adult.

PATCH
Use this to update an Adult. Provide the Adult as json in the request body; all fields will be updated. Returns the updated Adult.
Example:
  • .../Adults/5

May return 400 Bad Request, if the data is invalid.
May return 404 Not Found, if an Adult with the specified ID was not found.


Interest

The format of an interest is just a string, when sending back and forth. However, the Interest of a child is a class, containing a string.
API Request Sub-path Description Exceptions
/Interests GET
Use this to get a collection of all interests.
Example:
  • .../Interests
 May return 500 Internal Server Error, if something fails.
PUT /{interest:string}
Use this to create a new type of Interest. Provide the interest in the URL - spaces in URL are either - or %20
Example:
  • .../Interests/Foosball
  • .../Interests/Chess
  • .../Interests/Gaming
  • .../Interests/Jumping-Jacks
 May return 400 Bad Request, if the request did not succeed.

Pet

The Pet is sent back and forth as json. A new Pet can only be created through a Family as a Family-pet, or through a Child in a Family, as a Child-pet. Thus, there is no POST or PUT request available for Pets.
The format of a Pet is seen in the example below: 
{
    "ID": 0,
    "Species": "Cat",
    "Name": "Cassie",
    "Age": 6
}
API Request Sub-path Description Exceptions
/Interests GET
Use this to get a collection of Pets.
This request takes an option number of the following request parameters:
  • species - spaces in URL are either - or %20
  • age
Example:
  • .../Pets?species=cat
  • .../Pets?species=dog&age=4
  • .../Pets?age=2
  • .../Pets?species=guinea-pig
 May return 500 Internal Server Error, if something fails.
DELETE /{id:int}
Use this to delete a Pet. Provide the id in the URL. Alternatively the id can be provided through the query. The deleted Pet is returned. Example:
  • .../Pets/3
  • .../Pets?id=2
 May return 400 Bad Request, if the request did not succeed.

Child

The Child is sent back and forth as json.
The format of a Child is seen in the example below. A child has a collection of ChildInterests, and a collection of Pets. 
{
    "ChildInterests": [
        {
            "InterestId": "Handball"
        },
        {
            "InterestId": "Mountain Biking"
        },
        {
            "InterestId": "Bowling"
        }
    ],
    "Pets": [
        {
            "ID": 0,
            "Species": "Rat",
            "Name": "Twinkle",
            "Age": 4
        }
    ],
    "ID": 0,
    "FirstName": "Isabella",
    "LastName": "Warren",
    "HairColor": "Brown",
    "EyeColor": "Brown",
    "Age": 17,
    "Weight": 47.940002,
    "Height": 151,
    "Sex": "F"
},
API Request Sub-path Description Exceptions
/Children GET /{id:int}
Use this to get a single Child.
The id is provided through the URL. Alternatively the id is provided through the query. Example:
  • .../Children/3
  • .../Children?id=2
 May return 404 Not Found, if a Child with the provided ID is not found.
Use this to get a collection of Children.
This request takes an option number of the following request parameters:
  • firstname
  • lastname
  • haircolor
    valid values: [brown, blond, red, black, white, grey, blue, green, leverpostej]
  • eyecolor
    valid values: [brown, grey, green, blue, amber, hazel]
  • sex:F/M
  • age
  • id
  • interests - spaces in URL are either - or %20
The interests parameter is a list, so you can specify multiple interests, see example below.
Example:
  • .../Children?lastname=johnson&haircolor=blond
  • .../Children?interests=soccer&interests=bowling
 May return 500 Internal Server Error, if something fails.
PUT
Use this to add a Child. Provide the Child as json in the request body. The created Child is returned.
Example:
  • .../Children
 May return 400 Bad Request, if the Child is not valid.
May return 500 Internal Server Error, if the request failed.
DELETE /{id:int}
Use this to delete a Child. Provide the id in the URL. Alternatively the id can be provided through the query. The deleted Child is returned. Example:
  • .../Children/3
  • .../Children?id=2
 May return 404 Not Found, if a Child with the provided id was not found. May return 500 Internal Server Error, if something fails.

API Request Sub-path Description Exceptions
/Families GET
Use this to get a Collection of families.
This request takes an option number of the following request parameters:
  • streetname - spaces in URL are either - or %20
  • housenumber
  • numberOfChildren
  • numberOfParents
Example:
  • .../Families?streetname
 May return 500 Internal Server Error, if something failed.
PUT
Use this to create a new Family. Provide the Family as json in the request body.
Example:
  • .../Families
 May return 400 Bad Request, if the provided Family is invalid. May return 500 Internal Server Error, if something failed.
DELETE
Use this to delete a Family. The deleted family is returned. You must provide two query parameters:
  • streetname
  • housenumber
Example:
  • .../Families?streetname=farvergade&housenumber=3
 May return 400 Bad Request, if the parameters are invalid. May return a 404 Not Found, if the a Family was not found at the provided adress. May return 500 Internal Server Error, if something failed.
